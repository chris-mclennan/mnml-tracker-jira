# Cutting a release

This repo uses [`cargo-dist`](https://opensource.axo.dev/cargo-dist/)
for cross-platform binaries + a sibling Homebrew tap for `brew
install chris-mclennan/tap/mnml-tracker-jira`.

## First-time setup (one-time)

The release machinery is wired in `dist-workspace.toml` +
`.github/workflows/bump-homebrew-tap.yml`. The third piece —
`.github/workflows/release.yml` — is generated by cargo-dist:

```sh
# Install cargo-dist (one-time, you only need this on your dev machine)
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/axodotdev/cargo-dist/releases/download/v0.32.0/cargo-dist-installer.sh | sh

# Generate the release workflow
dist init --yes

# Verify it built the expected files
ls .github/workflows/release.yml dist-workspace.toml
```

`dist init` reads `dist-workspace.toml`, writes a `release.yml`
that builds for the 6 platforms configured there, and respects
the `github-build-setup` snippet at `.github/build-setup.yml`
(which clones the path-dep siblings on the runners). Re-run
`dist init` whenever you upgrade the `cargo-dist-version` in
`dist-workspace.toml`.

Add the `HOMEBREW_TAP_TOKEN` secret to this repo's settings (a
fine-grained PAT with **Contents: Read-and-write** on
`chris-mclennan/homebrew-tap`) — that's what
`bump-homebrew-tap.yml` uses to push the formula update.

## Cutting a new version

```sh
# 1. Bump version in Cargo.toml + commit
cargo set-version 0.3.0       # or hand-edit
cargo build                    # regenerates Cargo.lock
git add Cargo.toml Cargo.lock
git commit -m "v0.3.0"

# 2. Tag + push
git tag v0.3.0
git push origin main --tags
```

Pushing the tag triggers `.github/workflows/release.yml`:

- Builds tarballs / installers for the 6 targets in
  `dist-workspace.toml`
- Publishes a GitHub Release with all artifacts + a
  `sha256.sum` file
- The Release publish event then fires
  `.github/workflows/bump-homebrew-tap.yml`, which checks out
  `chris-mclennan/homebrew-tap`, runs the bump script with
  `mnml-tracker-jira` as the app, and commits the updated
  `Formula/mnml-tracker-jira.rb`

Users can then:

```sh
brew install chris-mclennan/tap/mnml-tracker-jira
```

## Verifying locally before tagging

```sh
./scripts/dist-check.sh        # if mnml's dist-check ports over
# OR
dist build --target=aarch64-apple-darwin
dist build --target=x86_64-apple-darwin
```

(macOS CI minutes are not free — verify cross-target builds locally
when possible to save burn cycles.)
